## Introduction to Reverse Engineering

Reverse engineering is the process of analyzing a system to understand its components, functionality, and behavior -- often without access to source code. In cybersecurity, we use reverse engineering to:

- Analyze malware and understand its capabilities
- Find vulnerabilities in closed-source software
- Crack software protections for educational purposes
- Understand proprietary protocols and file formats
- Solve CTF challenges in the Reverse Engineering category

## Computer Architecture Refresher

Before diving into reverse engineering, you need a solid understanding of how computers execute programs.

### CPU Registers (x86-64)

Modern x86-64 processors have several key registers:

| Register | Purpose | Notes |
|----------|---------|-------|
| RAX | Accumulator | Return values, arithmetic |
| RBX | Base | General-purpose, preserved |
| RCX | Counter | Loop counters |
| RDX | Data | I/O operations, arithmetic |
| RSI | Source Index | String operations, 2nd argument |
| RDI | Destination Index | String operations, 1st argument |
| RSP | Stack Pointer | Points to top of stack |
| RBP | Base Pointer | Stack frame base |
| RIP | Instruction Pointer | Next instruction address |

### The Stack

The stack is a LIFO (Last In, First Out) data structure used for:

- Passing function arguments
- Storing local variables
- Saving return addresses
- Managing function call frames

```asm
; Function prologue (enter a function)
push rbp          ; Save old base pointer
mov rbp, rsp      ; Set new base pointer
sub rsp, 0x20     ; Allocate 32 bytes for locals

; Function epilogue (leave a function)
leave             ; mov rsp, rbp; pop rbp
ret               ; Return to caller
```

## Essential Tools

### Ghidra

Ghidra is a free, open-source reverse engineering framework developed by the NSA. It includes:

- **Disassembler**: Converts machine code to assembly
- **Decompiler**: Attempts to reconstruct C source code
- **Debugger**: Analyze program execution
- **Script Engine**: Automate analysis with Python or Java

Here is how to create a basic Ghidra project:

```bash
# Download and extract Ghidra
wget https://github.com/NationalSecurityAgency/ghidra/releases/download/Ghidra_11.0_build/ghidra_11.0_PUBLIC_20231222.zip
unzip ghidra_11.0_PUBLIC_20231222.zip
cd ghidra_11.0_PUBLIC

# Run Ghidra
./ghidraRun
```

### GDB (GNU Debugger)

GDB with GEF (GDB Enhanced Features) or Pwndbg is essential for dynamic analysis:

```bash
# Install GDB
sudo apt install gdb

# Install GEF (GDB Enhanced Features)
bash -c "$(curl -fsSL https://gef.blah.cat/sh)"

# Or install Pwndbg
git clone https://github.com/pwndbg/pwndbg
cd pwndbg && ./setup.sh
```

Common GDB commands:

```gdb
# Basic commands
break *0x401000     # Set breakpoint at address
run                 # Run the program
continue            # Continue execution
stepi               # Step one instruction
nexti               # Step over call instructions
info registers      # Show register values
x/10gx $rsp         # Examine 10 giant words (8 bytes) from stack
disassemble main    # Disassemble main function
```

## Analyzing a Simple Binary

Let's analyze a simple program that checks a password:

```c
// challenge.c - The original source (unknown to the analyst)
#include <stdio.h>
#include <string.h>

int check_password(char *input) {
    char *correct = "s3cr3t_p4ss!";
    return strcmp(input, correct) == 0;
}

int main() {
    char buffer[64];
    printf("Enter password: ");
    scanf("%63s", buffer);
    
    if (check_password(buffer)) {
        printf("Access granted! Flag: CTF{r3v3rs3d!}\n");
    } else {
        printf("Access denied!\n");
    }
    return 0;
}
```

### Static Analysis with Ghidra

When we load this binary into Ghidra and analyze it, the decompiler might show something like:

```c
// Decompiled check_password function
bool check_password(char *param_1) {
    int iVar1;
    iVar1 = strcmp(param_1, "s3cr3t_p4ss!");
    return iVar1 == 0;
}
```

This reveals the hardcoded password, giving us the flag without ever running the program.

### Dynamic Analysis with GDB

Alternatively, using GDB to dynamically analyze the binary:

```gdb
# Load the binary
gdb ./challenge

# Disassemble main to find interesting addresses
(gdb) disassemble main
   0x0000000000401182 <+0>:     push   %rbp
   0x0000000000401183 <+1>:     mov    %rsp,%rbp
   ...

# Set breakpoint before strcmp
(gdb) break *0x4011a0

# Run and examine arguments
(gdb) run
Enter password: test
Breakpoint 1, ...
(gdb) x/s $rdi    # First arg to strcmp (our input)
0x7fffffffdc40: "test"
(gdb) x/s $rsi    # Second arg to strcmp (correct password)
0x402010:         "s3cr3t_p4ss!"
```

## Common Reverse Engineering Techniques

### String References

One of the easiest starting points is looking at strings in the binary:

```bash
# Extract all ASCII strings
strings challenge | less

# Search for interesting strings
strings challenge | grep -i "password\|flag\|correct\|denied"
```

### Cross-References in Ghidra

Once you find an interesting string or function, use cross-references (xrefs) to understand how it is used:

1. Find a string like "Access granted!"
2. Right-click and select "References" -> "Show References to Address"
3. Navigate to each reference to understand the code flow

### Function Identification

Common library functions have recognizable patterns. Learning to identify them by their behavior speeds up analysis:

- `strcmp` / `strncmp`: Compares two strings, usually before conditional jumps
- `malloc` / `calloc`: Allocates memory, return value stored in RAX
- `printf` / `puts`: Output strings to stdout
- `read` / `fgets`: Reads input from files or stdin

### Patching Binaries

Sometimes the easiest solution is to patch the binary to change its behavior:

```python
# Patch a binary to always succeed at password check
import sys

with open(sys.argv[1], 'rb') as f:
    data = bytearray(f.read())

# Change JE (74) to JNE (75) at the password check
# This reverses the logic: wrong passwords become correct
# Original: 74 0E  (je +14)
# Patched:  75 0E  (jne +14)
offset = 0x1234  # Address of the conditional jump
if data[offset] == 0x74:
    data[offset] = 0x75

with open(sys.argv[1] + '.patched', 'wb') as f:
    f.write(data)

print("[+] Binary patched!")
```

## Advanced Topics to Explore

Once you have mastered the basics, here are topics to deepen your skills:

- **Obfuscation Techniques**: Packing, control flow flattening, string encryption
- **Anti-Debugging**: Techniques programs use to detect debuggers
- **Binary Diffing**: Comparing patched binaries to find vulnerability fixes
- **Symbolic Execution**: Using tools like angr to automatically find paths to desired states
- **Firmware Analysis**: Extracting and analyzing embedded system firmware

## Conclusion

Reverse engineering is a skill that develops over time. Start with simple crackmes from platforms like Crackmes.one, and gradually work your way up to analyzing real-world malware and complex CTF challenges. The key is consistent practice and curiosity about how things work under the hood.
